\# CLAUDE.md



\## Project classification / プロジェクト分類



このプロジェクトは研究・ラボ用プロジェクトです。



以下の内容を混ぜないでください。



\- 個人趣味開発

\- 業務・商用プロジェクト

\- 本番環境の秘密情報

\- 顧客データ

\- 関係ない実験コード



\## Basic rules / 基本ルール



\- 明示的に依頼されるまで実装しないでください。

\- ファイルを変更する前に、必ず変更計画を説明してください。

\- ファイルを変更する前に、変更予定ファイルを列挙してください。

\- 依頼範囲外のファイルを変更しないでください。

\- 関係ない機能を追加しないでください。

\- 明示的な依頼なしに大規模リファクタをしないでください。

\- 既存テストを削除・緩和しないでください。

\- エラーを消すためだけに `any` や危険なキャストで逃げないでください。

\- エラーを黙って握りつぶさないでください。

\- 小さくレビューしやすい変更を優先してください。



\## Security rules / セキュリティルール



以下を読まない・変更しない・出力しない・要約しない・コミットしないでください。



\- `.env`

\- `.env.\*`

\- credentials

\- secrets

\- private keys

\- API keys

\- production logs

\- customer data

\- deployment credentials



明示的な指示なしに、本番環境に関わるコマンドを実行しないでください。



\## AI task workflow / AIタスク運用



実装指示書は以下に置きます。



`.ai/tasks/`



変更後は、以下に変更メモを書いてください。



`.ai/changes/`



変更メモには以下を含めてください。



\- 使用したタスクファイル

\- 変更したファイル

\- 各変更の理由

\- 実行したコマンド・テスト

\- 受け入れ条件を満たしたか

\- 残っている懸念

\- ロールバック方法

\- Cursorでレビューすべき観点



\## Review workflow / レビュー運用



Cursorでは以下を確認します。



\- タスクファイル

\- 変更メモ

\- git diff



Cursorレビューで指摘が出た場合、Claude Codeは最小差分で修正してください。
